学习如何使用Github和Git

一、关于Github

1、基本概念

  • 仓库（Repository）

    即项目的意思，想在github上开源一个项目，就必须要新建一个Reponsitory，仓库用来存放项目代码，每个仓库存放一个项目。

  • 收藏（Star）

    仓库主页Star按钮，意思是收藏项目的人数，收藏别人的项目是需要进入他人的项目主页。

    ![your stars](D:\仰望星空\Github\your stars.jpg)

  • 复制or克隆项目（Fork）

    即项目的分支（源于开源），最后可将分支合并拉取，并能看到每个分支的情况，比如代码修改量、修改人、修改时间等。fork后在分支方会产生一个与主项目相同的独立项目，但是主项目发生变化时，分支项目不变化。

  • 发起请求（Pull Ruquest）

    基于fork的项目分支作了改进后，想把改进合并到主项目里，就需要发起请求（Pull Request），主项目主收到请求后会review分支的代码，测试觉得OK后，会接受这个PR，就会把分支合到主项目里。

    发起新的pull ruquest请求：

    ![pull request](D:\仰望星空\Github\pull request.jpg)

  • 关注（Watch）

    Watch的项目如果有斜体更新，都会第一时间收到关于该项目的通知提醒。

    当被关注的人的项目有变化时：

  • 事务卡片（Issue）

    发现代码Bug，或有其他测试问题需要讨论时用。

  • Github主页

    该页左侧主要显示用户动态以及关注用户或关注仓库的动态，右侧显示所有的git库。

  • 仓库主页

    主要显示项目的信息，如项目代码、版本、收藏/关注/fork情况等。

  • 个人主页**

    个人信息等。

2、注册(sign up)及登录(sign in)github帐号

github官方网址

邮箱非常重要，需要接收邮件！

注意：

  1. github服务器在国外，有访问太慢或无法访问时需要 科学。
  2. github search (google helper、ssr等)。
  3. 私有仓库只能自己或指定的朋友才有权操作（但是收费，所说github被微软收购后私有仓库也免费了）。
  4. QQ邮箱需要设置白名单才可以收到确认邮件。

3、创建仓库

  • 一个仓库对应一个开源项目
  • 通过git管理这个库
  1. 仓库名称 Repository name（一般使用英文，并使用项目名称）
  2. 项目描述 Description(option可用中文)
  3. 创建一个Readme文件 Initialize this repository with a README（需要用markdown）

  • 项目主页里可以Edit项目描述

  • Create new file创建文件

  • Upload files上传文件

  • Find file搜索文件

4、仓库管理

  • 创建新文件（creat new file）

    • 起名\写代码\Commit new file写说明和详细说明
    • preview预览

  • 上传新文件（upload files）支持选择和拖曳文件

  • 查找文件（支持即输即显）

  • 编辑文件

  • 删除文件（信息可以在commits里查看）

  • commits显示对文件的处理次数

  • 下载/检出项目（clone or download，git下载用内部的地址）

6、开源项目贡献流程

  1. 方式1：新建Issue

    对项目主提交使用问题或建议和想法。

  2. 方式2：Pull Request

    直接修改、完善主项目的代码。

    • fork项目
    • 修改自己仓库的项目代码
    • 新建Pull Ruquest
    • 等待项目主审核

3、Git工作流程

4、Git初始化及仓库创建和操作

  1. 基本信息设置

    //第1步：在主文件夹(例如new)里右击，然后点击Git Bash Here进入命令行模式
//第2步：设置用户名
git config --global user.name 'github的用户名'

//第3步：设置用户邮箱
git config --global user.email 'XXX@XXX.com'
//以上的设置会在github主页显示信息
//如果选择Git GUI Here即进入图形化界面模式

  1. 初始化一个新的Git仓库

    • 在主文件夹new里创建新的文件夹，例如命名为test

    • cd test\git init或者在test文件夹上右击选择“Git 在这里创建版本库”完成初始化之后，会在test文件夹下生成一个.git的隐藏文件。

      ​

    1. 向仓库中添加文件

      //第1步：先在工作区里创建文件
//命令行模式
touch XXX.XX  //新建一个文件
//窗口模式，使用用传统windows操作模式
//可以先查看文件状态
git status
//第2步：从工作区向暂存区添加文件
git add XXX.XX
//这时用git status命令查看文件状态，就会看到文件状态和颜色会发生变化     
git status
//第3步：从暂存区向仓库添加文件
git commit -m"提交描述"
//再查看文件状态
git status //这时会发现nothing to commit,working directory clean

      4.修改文件

      // 第1步：修改文件需要键入下列命令
vi XXX.XX  #或者vim XXX.XX
//或者在网页上直接修改，修改后用下面的命令就能查看文件里的代码
cat XXX.XX
//这里再用git status命令查看就会发现
//modified XXX.XX (表示文件被修改过了)
//第2步：再把修改过的文件添加到仓库里就可以了
git commit -m"描述"

      5.删除文件

      //第1步：删除workshop工作区的文件,用命令行模式如下，或者在电脑里直接删除
rm -rf XXX.XX
//第2步：从暂存区删除文件，方式同上
git rm XXX.XX
//第3步：提交操作，同上
git commit -m"描述"

3. 解决git push 报错或提示没有权限问题

错误提示：The requested URl returned error:403 Forbidden while accessing

这是因为：私有项目，没有权限，输入用户名和密码，或者远程地址采用这种类型。

解决方法：

1. 代码方法
vi.git/config //重新初始化
//把[remote"origin"]url=https://github.com/用户名/仓库名.git
//修改为
//[remote"origin"]url=用户名:密码@github.com/用户名/仓库名.git

  1. 图形化界面方法

    第1步：进入.git这个隐藏文件夹里

    第2步：用记事本打开config文件

    第3步：按代码方法的字符修改
