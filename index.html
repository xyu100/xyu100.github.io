学习如何使用Github和Git

学习如何使用Github和Git

一、关于Github

1、基本概念

  • 仓库（Repository）

    即项目的意思，想在github上开源一个项目，就必须要新建一个Reponsitory，仓库用来存放项目代码，每个仓库存放一个项目。

  • 收藏（Star）

    仓库主页Star按钮，意思是收藏项目的人数，收藏别人的项目是需要进入他人的项目主页。

    ![your stars](D:\仰望星空\Github\your stars.jpg)

  • 复制or克隆项目（Fork）

    即项目的分支（源于开源），最后可将分支合并拉取，并能看到每个分支的情况，比如代码修改量、修改人、修改时间等。fork后在分支方会产生一个与主项目相同的独立项目，但是主项目发生变化时，分支项目不变化。

  • 发起请求（Pull Ruquest）

    基于fork的项目分支作了改进后，想把改进合并到主项目里，就需要发起请求（Pull Request），主项目主收到请求后会review分支的代码，测试觉得OK后，会接受这个PR，就会把分支合到主项目里。

    发起新的pull ruquest请求：

    ![pull request](D:\仰望星空\Github\pull request.jpg)

  • 关注（Watch）

    Watch的项目如果有斜体更新，都会第一时间收到关于该项目的通知提醒。

    当被关注的人的项目有变化时：

  • 事务卡片（Issue）

    发现代码Bug，或有其他测试问题需要讨论时用。

  • Github主页

    该页左侧主要显示用户动态以及关注用户或关注仓库的动态，右侧显示所有的git库。

  • 仓库主页

    主要显示项目的信息，如项目代码、版本、收藏/关注/fork情况等。

  • 个人主页**

    个人信息等。

2、注册(sign up)及登录(sign in)github帐号

github官方网址

邮箱非常重要，需要接收邮件！

注意：

  1. github服务器在国外，有访问太慢或无法访问时需要 科学。
  2. github search (google helper、ssr等)。
  3. 私有仓库只能自己或指定的朋友才有权操作（但是收费，所说github被微软收购后私有仓库也免费了）。
  4. QQ邮箱需要设置白名单才可以收到确认邮件。

3、创建仓库

  • 一个仓库对应一个开源项目
  • 通过git管理这个库
  1. 仓库名称 Repository name（一般使用英文，并使用项目名称）
  2. 项目描述 Description(option可用中文)
  3. 创建一个Readme文件 Initialize this repository with a README（需要用markdown）

  • 项目主页里可以Edit项目描述

  • Create new file创建文件

  • Upload files上传文件

  • Find file搜索文件

4、仓库管理

  • 创建新文件（creat new file）

    • 起名\写代码\Commit new file写说明和详细说明
    • preview预览

  • 上传新文件（upload files）支持选择和拖曳文件

  • 查找文件（支持即输即显）

  • 编辑文件

  • 删除文件（信息可以在commits里查看）

  • commits显示对文件的处理次数

  • 下载/检出项目（clone or download，git下载用内部的地址）
